# ADR-0002 三层授权标签:每个节点和边出生即带 tier

- 状态:Accepted(2026-06-10)
- 来源:candy 划定的授权红线(2026-06-10 brainstorm)

## 决策

每个 node / edge 必须带 `tier: "shared" | "private" | "aggregate"` 字段,schema 强制必填。导出共有包 = 按 tier **物理过滤**(过滤后数据中私有记录不存在),不是视图遮罩。

| tier | 内容 | 去向 |
|---|---|---|
| shared | 行业/产污场景/污染源/污染物/标准/技术规范/法条/问题分类法 | 软著、培训、执法工具共有包 |
| private | 证据标准/整改库/报告表达/企业数据(脱敏也不出) | 永不导出;演示"看得见带不走" |
| aggregate | 区域/行业统计(最小样本 ≥5 家) | 单向输出做政策支持 |

## 理由

软著与政府共建要求一部分资产公开,商业壁垒要求另一部分永不出门——两者用同一张图。事后补打标签不可行(无法证明历史导出未泄漏),必须出生即带。

## 后果

- `pipeline/validate_no_private_leak.py` 是拒绝型契约测试,任何导出动作的前置门禁。
- 共有导出包带 manifest + hash,可审计。
- 禁止实现"例外白名单"。tier 修改属 HIGH 风险,需 candy 批准。
