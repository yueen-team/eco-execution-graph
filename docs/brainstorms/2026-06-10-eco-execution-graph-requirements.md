# 环保现场执行图谱 · 需求文档

- 日期:2026-06-10
- 来源:CE brainstorm(Deep — product 级,五个严谨性缺口全部探查完毕)
- 状态:candy 已确认范围(Phase 2.5 scoping synthesis 通过)

## Summary

「环保现场执行图谱」第一刀:危废单域打穿——5 张精品《法条↔现场执行卡》+ 15 张半成品卡 + 双向缺口报告作为内容核心,交互式 ego 图谱 + 真实蒸馏计数器离线快照作为呈现外壳,演示叙事设计成"给监控中心执法大队工具补上现场那一半"。所有节点、边、source 从出生起带三层授权标签(共有/私有/聚合),同一张图谱同时服务软著共有包和后续回灌 EcoCheck。

## Problem Frame

环保管家公司每月服务约 100 家企业,EcoCheck 小程序每天蒸馏出新的现场问题、审核经验、整改经验——这是行业里任何大厂和政府都生产不了的资产,但过去没有被结构化为可复用、可引用、可输出的图谱。

同时,生态环境监控中心主任(AI 技术行家)主动提出合作:他们建法典知识库,优势是法律法规文本与案例;我们优势是真实现场经验。他们正在给基层执法大队做工具,"有法律依据但脱离真实企业场景"——正好是我们图谱的镜像缺口。他们进度慢,第一个把"拼接样板"摆上桌的人定义整个合作的形态和标准。

战略本质:用没人愿意付的成本(精力/时间/token),买下"云南省环保现场服务行业标准制定者"的位置。

## Requirements

- **R1** 危废域第一轮样板:5 张精品《法条↔现场执行卡》打穿全链路,15 张半成品卡证明可规模化;卡片链路为法条瘦引用 → 现场表现 → 证据类别/字段要求 → 整改方向 → 报告表达约束 → 踩雷点。
- **R2** 双向缺口报告:法条在现场零覆盖清单(合规盲区)+ 现场问题无法条依据清单(管理经验 vs 法律要求的强制区分)+ 高频踩雷条款排行。
- **R3** 交互式 ego 图谱 UI:从任一节点(法条/问题/行业/企业类型)点开辐射邻居,按边类型筛选、按 confidence 和审核状态着色、节点带三层授权徽章。
- **R4** 实时蒸馏计数器:演示界面展示"本月新增现场经验 N 条",证明图是活的。
- **R5** 三层授权标签:每个节点和每条边出生即带 `tier: shared | private | aggregate`,共有包导出是按标签过滤的物理隔离。
- **R6** 演示"看得见、带不走":私有层(证据标准/整改库/报告表达/企业数据)在演示中活生生展示,但在任何共有导出包中物理不存在。
- **R7** 法条瘦节点:图中法条节点只存 ID、条款号、义务谓词、`lineage_ref`(法典沿革指针,预留给政府数据);全文永远从腾讯云知识引擎 RAG / 政府法典库取。
- **R8** 踩雷点(法律与现实脱节点)为一等公民节点族:pitfall_class(shared)、pitfall_pattern_stat(aggregate)、pitfall_instance(private)。
- **R9** 演示叙事素材:按 C 刀法准备"执法工具升级前 vs 升级后"的对照展示。
- **R10** P0.5 上下文装配最小验证:合成企业 + 3 条危废问题 + 法条 ID 离线装配月报段落,由 ETO 对比旧 AI 与图谱装配版。
- **R11** 图谱质量评分:每条边必须给出 confidence_reason、evidence_count、last_verified_at、reviewer_role、staleness_risk。
- **R12** 云南环保高频踩雷地图:只用 aggregate 层输出区域/行业/环保维度/问题类型/法条规范/复发率/整改难度。
- **R13** 监管口径一致性检查器:输入报告结论,检查错引法条、管理经验违法化、证据链缺失、过度承诺、已废止/待确认条款和政府口径不一致。

## Key Decisions

| # | 决策 | 理由 |
|---|---|---|
| D1 | 先打政府侧演示,后做月报升级 | 监管背书反过来放大商业壁垒,顺序不可逆;对方主动邀约+进度慢=窗口在我方 |
| D2 | 深度优先:只做危废一个域 | 一条打穿全链的竖切胜过 13 个维度各做一半(与 2026-06-01 审计结论一致) |
| D3 | B 骨 + A 壳 + C 刀法组合 | 卡片集=软著与标准实体;ego 图=技术发烧友主任的第一眼震撼;"补全他们的执法工具"=最高级的震撼方式 |
| D4 | 软著第一创作人让给监控中心 | 换"与主管部门统一口径"——让我方问题分类法成为带官方背书的行业口径 |
| D5 | 赌注不押政府背书 | 有没有政府支持都要做到行业顶尖;护城河=每天 100 家企业的蒸馏流,背书只是放大器 |
| D6 | 成功标准是关系性的 | 政府侧认可+正面形象+愿意在企业面前提及/推荐,而非技术指标 |

## Scope Boundaries

**第一刀不包含(第二刀及以后):**
- 月报生成正式接入和 EcoCheck 运行时回灌(演示成功后做,即 EcoDoc M3 接地);但 P0.5 先做离线最小验证
- 其他 12 个环保维度的铺开
- 与云南核算器的系数桥接(蒸馏 v2 spec 的 accounting_bridge 钩子保留,本期不消费)
- 图数据库(Neo4j 等)基建
- 政府 lineage 数据的实际接入(接口契约先行,不阻塞)

**永久边界(授权红线):**
- 可出去(shared):行业画像、产污场景、污染源、污染物、标准、技术规范、法规法条、问题分类法
- 打死不出(private):证据标准、整改建议库、报告表达模板、单个企业数据(脱敏也不出)
- 聚合可出(aggregate):区域级/行业级统计规律,做政策与数据支持;由私有层计算、单向输出

shared 层是口径层/骨架层/统计层,不是完整产品层。完整能力留在 private + internal runtime。

## Success Criteria

1. 主任团队看到演示后的反应达到"震惊和撼动"级,认可"这就是我要的现场数据"。
2. ETO 认可图谱装配后的月报段落明显优于普通 AI 直接写。
3. 政府侧赢得认可与正面形象,愿意在企业面前提及甚至推荐。
4. 软著推进(我方第二创作人)且口径统一落地。
5. 同一张图谱无需改造即可支撑第二刀(月报上下文装配)。
6. 共有导出包经契约测试证明:私有层零泄漏。
7. 14 天危废精品切片能同时证明政府演示、ETO 认可、月报变好、私有层安全和可扩域。
